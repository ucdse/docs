
# Backend

In terms of backend architecture implementation, when I was developing the **journey planner API**, a simplistic initial logic for querying available bikes unexpectedly caused a server CPU overload and crash. After presenting the error logs and my SQLAlchemy query snippet to the AI, it quickly identified an out-of-memory deadlock triggered by an infinite full table scan on the `Availability` table. It assisted me in applying an implicit bounding time window filter tracing back 1 hour, which resolved the issue. Furthermore, during the backend testing setup, the AI provided extremely detailed guidance on how to use `unittest.mock` to intercept and mock the return values of the Google Maps API, ensuring that every **Pytest test run** achieved 100% determinism in isolated, offline conditions.

Building upon this, when I needed to integrate the core **machine learning backend prediction workflow** with the **AWS RDS** database, I encountered severe cold start latency. My initial design was based on "lazy loading": only when a real-time request was received from the frontend would the backend dynamically extract a massive amount of data to load into the machine learning model for inference. This resulted in an extremely long response blockage during the first API call. I discussed the prediction routing structure with the AI for potential solutions. It helped me analyze the deployment environment and proposed a deep refactoring architecture—utilizing the flask App context lifecycle for "Pre-warming" and cleverly applying copy-on-write at the memory layer. With this optimization, my backend ultimately managed to efficiently and instantaneously feed massive prediction features into the machine learning model while ensuring that prediction results were returned to the frontend without any stutter.

# Frontend

Regarding frontend engineering and interactive design, my main focuses were the complete rendering of the **journey planner UI** and the **chat page**. While completing the multi-modal route navigation query, the AI assisted me in building a responsive loop based on `google.maps.places.Autocomplete`. It also cleared up frontend state management from a TypeScript type perspective, seamlessly rendering the concurrent overlay of a "walk-cycle-walk" three-segment multi-color route, along with wrapping an automatic `fitBounds` zoom for an intelligent viewpoint. As for the LLM inference-based AI chat page development, the `EventSource` API I originally used was incompatible with the project's secure JWT Token fetching. The AI provided a deep refactoring solution: rebuilding the network layer using the underlying `fetch` and `ReadableStreamDefaultReader` to capture the model's byte stream. This flawlessly achieved a Server-Sent Events (SSE) typewriter-style word-by-word streaming effect with identity authentication. Built on this foundation, combining the AI-suggested local state interception, I also managed to persist the session state across cross-page reloads.
